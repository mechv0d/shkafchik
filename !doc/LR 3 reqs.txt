Интеграция Redux Toolkit в существующий проект
Цель:
Научиться работать с Redux Toolkit для управления глобальным состоянием приложения. 

1. Подготовка проекта:
• Убедитесь, что ваш существующий React проект работает корректно
• Установите Redux Toolkit: npm install @reduxjs/toolkit react-redux
• Установите Redux DevTools Extension в браузер
• Проанализируйте, какое состояние в вашем проекте можно перенести в Redux

2. Настройка store:
• Создайте папку store/ или redux/ в src/
• Создайте файл store.ts с помощью configureStore
• Настройте Provider в корневом компоненте приложения (main.tsx или App.tsx)
• Убедитесь, что Redux DevTools подключены и работают

3. Создание slices:
• Создайте минимум 2-3 slice для разных частей вашего приложения
• Примеры: userSlice, cartSlice, productsSlice, authSlice и т.д. (в зависимости от вашего проекта)
• Используйте createSlice для каждого среза состояния
• Определите initialState для каждого slice
• Создайте reducers для синхронных действий

4. Реализация синхронных действий:
• Создайте минимум 5-7 reducers в ваших slices
• Примеры действий: добавление/удаление элемента, изменение данных, сброс состояния
• Экспортируйте actions из каждого slice
• Убедитесь, что reducers используют иммутабельные обновления (благодаря Immer)

5. Асинхронные операции с createAsyncThunk:
• Создайте минимум 2-3 thunk для асинхронных операций
• Примеры: загрузка данных с API, отправка формы, удаление записи
• Настройте обработку состояний: pending, fulfilled, rejected
• Используйте extraReducers для обработки этих состояний
• Добавьте поля loading и error в state для отображения статуса

6. Интеграция с компонентами:
• Замените локальный useState на Redux в минимум 3-5 компонентах
• Используйте useSelector для получения данных из store
• Используйте useDispatch для отправки actions
• Создайте кастомные хуки для упрощения доступа к Redux (например: useCart, useAuth)

7. Структурирование кода:
• Организуйте структуру: store/slices/, store/thunks/, store/hooks/
• Создайте типы TypeScript для вашего state (если используете TS)
• Экспортируйте типы RootState и AppDispatch из store
• Вынесите общие селекторы в отдельные функции

8. Оптимизация селекторов:
• Создайте минимум 2 мемоизированных селектора с помощью createSelector
• Избегайте создания новых объектов/массивов в селекторах
• Используйте селекторы для вычисляемых значений (например: totalPrice, filteredItems)

9. Redux DevTools и отладка:
• Проверьте работу всех actions через Redux DevTools
• Используйте Time Travel для отладки состояний
• Убедитесь, что все actions имеют понятные имена
• Проверьте, что state обновляется корректно

10. Обработка ошибок:
• Реализуйте обработку ошибок в thunks
• Добавьте отображение ошибок в UI
• Создайте механизм сброса ошибок
• Протестируйте поведение при сетевых ошибках

11. Тестирование функциональности:
• Проверьте работу всех CRUD операций через Redux
• Убедитесь, что состояние сохраняется между переходами по страницам
• Проверьте корректность работы асинхронных операций
• Протестируйте edge cases (пустые списки, ошибки API и т.д.)

12. Git и документация:
• Создайте отдельную ветку: feature/redux-toolkit-integration
• Делайте коммиты по шагам: setup, slices, thunks, components, optimization
• Напишите README с описанием структуры Redux в проекте
• Опишите какие части состояния управляются через Redux

13. Отчет о выполнении:
• Создайте отчет в PDF формате с описанием проделанной работы
• Включите скриншоты Redux DevTools с вашими actions и state
• Опишите архитектуру Redux в вашем проекте (какие slices создали и зачем)
• Добавьте примеры кода: минимум 1 slice и 1 thunk
• Сравните 'до' и 'после' интеграции Redux (преимущества и сложности)
• Добавьте ссылку на ваш репозиторий GitHub/GitLab

14. Критерии оценки:
• Корректная настройка store и Provider
• Создание минимум 2-3 slices с reducers
• Реализация минимум 2-3 асинхронных thunks
• Интеграция Redux в минимум 3-5 компонентов
• Использование мемоизированных селекторов
• Качество структуры кода и организация файлов
• Обработка ошибок и состояний загрузки
• Наличие подробного отчета с примерами кода и скриншотами
